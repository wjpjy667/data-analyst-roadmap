    🔍第零步：先做调查（第1-3天）

    不要一上来就学SQL。先搞清楚市场到底要什么，再决定怎么学。
    #任务时间产出
    1浏览BOSS直聘/拉勾，阅读杭州100个数据分析岗位JD，标注共性技能要求2-3h技能需求清单
    2找出出现频率最高的5个技能关键词，排优先级1h优先级排序表
    3在脉脉/知乎/即刻上找3个在职数据分析师，礼貌私信问他们实际工作内容2h3份对话记录
    4用调查结果重新审视学习路线，调优先级1h调整后的学习计划
    为什么这一步重要：你可能发现市场上最要的不是Python而是SQL+BI工具，或者发现数据运营比数据分析师更好入行。这些认知比盲目学三个月有价值得多。

    4. 数据运营和BI分析师是最适合民办本转行的两个入门岗位——门槛低、需求量大、不卡学历

    5. 杭州电商方向是最大就业市场，你不需要学金融/风控知识，专注电商数据分析即可

    6. 80%的JD要求「业务理解能力」——这只能靠做项目积累，没法纯学理论

    📚学习路线（项目从第1周开始）

    核心原则：学什么就做什么项目。每周至少产出一个可展示的东西。到第14周你有十几个作品，从中选最好的三个放进简历。
    阶段一 · SQL · 第1-4周
    周次学什么本周项目（必须产出）
    第1周SELECT、WHERE、GROUP BY、HAVING、聚合函数SQL查询集 #1：找公开数据集，写10条业务查询——日活、下单率、客单价、品类排名
    第2周多表联查：JOIN全家桶、UNIONSQL查询集 #2：建3个关联表，写5条多表联查——用户购买明细、品类交叉分析
    第3周子查询、CASE WHEN、窗口函数SQL查询集 #3：用户留存SQL、RFM分层SQL、连续登录天数SQL——至少5条，含窗口函数
    第4周综合实战SQL分析报告：自选数据集，从数据清洗到分析结论，用SQL完成全流程，写2页分析总结
    阶段二 · Excel + BI · 第5-6周
    第5周Excel：数据透视表、VLOOKUP、条件格式、图表Excel分析报告：拿SQL分析的数据，用Excel做带图表、带结论的分析报告
    第6周Power BI：Power Query清洗 + 可视化看板搭建Power BI看板：搭建完整业务看板（含至少5个图表），导出截图
    阶段三 · Python · 第7-12周
    7-8周Python基础 + Pandas核心操作Python数据处理脚本：读取CSV→清洗→分组聚合→导出结果
    9-10周Pandas进阶 + 数据清洗实战Python + SQL联合分析：SQL取数 → Python清洗分析 → 输出结论
    第11周Matplotlib + Seaborn 可视化可视化作品集：10张不同类型图表，每张配一句话结论
    第12周全流程综合完整分析项目：数据获取→SQL处理→Python分析→可视化→结论报告
    阶段四 · 打磨 · 第13-14周
    第13周从已有作品中选最好的3-5个，深度打磨3个精品项目：每个形成PDF报告（数据→分析→图表→结论），放GitHub + 语雀
    第14周简历包装 + 模拟面试最终简历：以项目经验为主体，突出"能独立完成数据分析全流程"

    📋面试投递节奏（每两周一次，获取市场反馈）

    时间动作目的
    第2周投10份简历测试市场反应，别等准备好了再投
    第4周投10份，更新简历根据反馈修正简历和技能方向
    第6周投15份，调整投递岗位类型对比不同岗位的反馈差异
    第8周投15份，更新简历（加了Python项目）看简历变强后反馈是否变好
    第10周投15份，主动约面试检验面试能力，积累真实面试经验
    第12周投15份，集中面试面试是免费市场调研——面试官问什么，市场就要什么
    第14周+集中投递，目标拿offer简历和面试能力已经过6轮迭代
    核心心态：面试不是为了通过，是为了获取信息。被拒了比没人回应好——至少你知道了市场在乎什么。每次面试回来更新学习优先级。

    Plan B 岗位为什么下一步
