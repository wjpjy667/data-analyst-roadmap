🚫为什么不做PLC/电气自动化

对比维度PLC / 电气数据分析结论

🔍第零步：先做调查（第1-3天）

不要一上来就学SQL。先搞清楚市场到底要什么，再决定怎么学。
#任务时间产出
1浏览BOSS直聘/拉勾，阅读杭州100个数据分析岗位JD，标注共性技能要求2-3小时技能需求清单
2找出出现频率最高的5个技能关键词，排优先级1小时优先级排序表
3在脉脉/知乎/即刻上找3个在职数据分析师，礼貌私信问他们实际工作内容2小时3份对话记录
4用调查结果重新审视学习路线，调优先级1小时调整后的学习计划
为什么这一步重要：你可能发现市场上最要的不是Python而是SQL+BI工具，或者发现数据运营比数据分析师更好入行。这些认知比盲目学三个月有价值得多。

📚学习路线（按周拆分，项目从第1周开始）

核心原则：学什么就做什么项目。每周至少产出一个可展示的东西。到第14周你有十几个作品，从中选最好的三个放进简历。
阶段一 · SQL 数据库查询 · 第1-4周
周次学什么重点本周项目（必须产出）
第1周SELECT、WHERE、GROUP BY、HAVING、聚合函数分组与聚合SQL查询集 #1：找一份公开数据集（如淘宝用户行为数据），写10条业务查询——日活、下单率、客单价、品类排名
第2周多表联查：JOIN全家桶、UNIONJOIN条件SQL查询集 #2：建3个关联表（用户、订单、商品），写5条多表联查——用户购买明细、品类交叉分析
第3周子查询、CASE WHEN、窗口函数（ROW_NUMBER、RANK、LAG/LEAD）窗口函数SQL查询集 #3：用户留存SQL、RFM分层SQL、连续登录天数SQL——至少5条，含窗口函数
第4周综合实战业务场景SQL分析报告：自选数据集，从数据清洗到分析结论，用SQL完成全流程，写一份2页的分析总结
阶段二 · Excel + BI工具 · 第5-6周
第5周Excel：数据透视表、VLOOKUP、条件格式、图表透视表Excel分析报告：拿第4周SQL分析的同一批数据，用Excel做一份带图表、带结论的分析报告
第6周Power BI：Power Query清洗 + 可视化看板搭建看板搭建Power BI看板：搭建一份完整业务看板（含至少5个图表），导出截图放到语雀/知乎
阶段三 · Python 数据分析 · 第7-12周
7-8周Python基础 + Pandas核心操作DataFrame操作Python数据处理脚本：读取CSV→清洗→分组聚合→导出结果。代码+结果截图发语雀
9-10周Pandas进阶 + 数据清洗实战groupby/merge/applyPython + SQL联合分析：SQL取数 → Python清洗分析 → 输出结论。构建完整的分析管道
第11周Matplotlib + Seaborn 可视化图表类型可视化作品集：10张不同类型图表，每张配一句话结论
第12周全流程综合端到端完整分析项目：从数据获取→SQL处理→Python分析→可视化→结论报告，全流程走通
阶段四 · 整合打磨 · 第13-14周
第13周从已有作品中选出最好的3-5个项目，深度打磨质量提升3个精品项目：每个项目形成一份PDF报告（数据→分析→图表→结论），放到GitHub + 语雀
第14周简历包装 + 模拟面试市场准备最终简历：以项目经验为主体，突出"能独立完成数据分析全流程"

🚀为什么要从第1周就做项目

旧方案（错）新方案（对）
先学13周，最后2周赶项目第1周就开始做项目，学什么做什么
简历上只有3个项目有14个作品，挑最好的3个放简历
项目很"学生气"每周积累的作品叠加起来，展示的是持续输出能力
面试被问项目经历时只能讲3个能讲14周的学习轨迹和项目迭代过程
发布平台：语雀（公开知识库）、知乎专栏、GitHub。把每周作品发出来，既是存档也是建立行业存在感。

📋面试投递节奏（不是为了找工作，是获取市场反馈）

时间动作目的
第2周投10份简历（不要等准备好了再投）测试市场反应，看有没有回音
第4周投10份，更新简历根据前两周的反馈修正简历和技能方向
第6周投15份，尝试调整投递的岗位类型对比不同岗位（数据分析 vs 数据运营）的反馈差异
第8周投15份，更新简历加了Python项目后简历变强了，看反馈是否变好
第10周投15份，开始主动约面试检验自己的面试能力，积累真实面试经验
第12周投15份，集中面试把面试当成免费市场调研——面试官问什么，市场就要什么
第14周+集中投递，目标拿offer此时的简历和面试能力已经经过6轮迭代
核心心态：面试不是为了通过，是为了获取信息。被拒了比没人回应好——至少你知道了市场在乎什么、不在乎什么。每次面试回来更新你的学习优先级。

🅱️ B计划：如果年底没入行

Plan B 岗位为什么下一步
数据运营技术门槛最低，偏业务，对项目经验要求不如数据分析师高干6-12个月，积累业务经验后内部转或跳数据分析岗
BI助理主要做报表和看板，SQL+Excel够用，不需要Python边干边学，积累1年BI经验后跳数据分析或BI分析师
电商运营（数据方向）杭州电商公司多，岗位量大，入行竞争小主动承担团队的数据工作，积累分析经验后转岗
关键心态：第一份工作是一张入行门票，不是终点。不管走哪条路进去，目标是第一年积累真正的分析项目经验。有了经验，学历和专业都不重要了。
